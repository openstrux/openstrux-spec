# Openstrux v0.6.0 — Formal Manifesto Review

Evaluated against `MANIFESTO_OBJECTIVES.md` and `MANIFESTO_BENCHMARKS.md`.

Artifact: spec + implementation (alpha). Version: 0.6.0. Date: 2026-03-22.

> **Alpha-grade review.** v0.6.0 is alpha: one LLM family, 10/20+ benchmark cases, generation comparison not yet run. All WARN-data ratings reflect incomplete measurement, not failing results. Full release gate at v0.7.0.

---

## What Changed Since v0.4-draft

- **TypeScript target generator** implemented (ADR-019, native package output). B001-B010 baseline TypeScript measured.
- **Benchmark suite B001-B010** written and token-measured with tiktoken cl100k_base.
- **Grant-workflow use case** created (`openstrux-uc-grant-workflow`): domain model, specs, prompts, unit tests, benchmark runner.
- **Config inheritance, panel shorthand, expression shorthand** all implemented in spec (carried from v0.4 blocking actions).
- **@dp, @access, compliance rods** (`pseudonymize`, `encrypt`, `validate`, `guard`) specified and implemented.
- **Benchmark runner** implemented (`openstrux/benchmarks/runner/`): generates code via API, runs tests, archives results.

## Alpha-Grade Summary

v0.6.0 delivers the first end-to-end working Openstrux system: spec, parser, validator, TypeScript generator, and a complete use case (grant-workflow). Three principles score PASS with full evidence; three score WARN-data reflecting planned measurements. No principle fails.

The blocking action from v0.4-draft ("Build the benchmark suite") is resolved: B001-B010 cases written and measured. The remaining WARN-data items are all measurement gaps, not design failures, and are deferred to v0.7.0 per the alpha scope defined in the release plan.
